23 数据可视化基础

好的图表胜过千言万语。本章介绍 ggplot2 的语法体系和科研论文级图表的制作方法。

数据来源：本章示例数据结合两个真实数据源： 1. Li et al. (2017) 喀斯特土壤碳氮研究 (DOI: 10.1002/2016JG003683) 2. Du et al. (2020) 广西木论保护区物种组成数据 (DOI: 10.11922/csdata.2019.0031.zh)

23.1 ggplot2 语法体系

ggplot2 基于 Wilkinson 提出的”图形语法”（Grammar of Graphics）理论（Wilkinson, 2005, The Grammar of Graphics, 2nd ed., Springer, DOI: 10.1007/0-387-28695-0），将图表分解为几个核心组件：

ggplot(data, aes(x, y)) +   # 数据和映射
  geom_xxx() +                # 几何对象（画什么）
  scale_xxx() +               # 标度（怎么映射）
  labs() +                    # 标签
  theme()                     # 主题（外观）
Noteaes() 是什么？

aes() 是 aesthetic mapping（美学映射）的缩写，它告诉 ggplot2 如何将数据变量映射到图形属性上：

  • aes(x = soil_ph) → 将 soil_ph 映射到 x 轴
  • aes(color = habitat) → 将 habitat 映射到颜色
  • aes(size = elevation) → 将 elevation 映射到点的大小

关键区别：写在 aes() 里面的属性由数据驱动（每个数据点可以不同），写在 aes() 外面的属性是固定值（所有点相同）。例如 geom_point(aes(color = habitat)) 按生境着色，而 geom_point(color = "red") 所有点都是红色。

Tip+ 运算符

ggplot2 使用 + 将图层逐层叠加。每个 + 后面添加一个新的组件（几何对象、标度、主题等）。注意 + 必须放在行末，不能放在下一行的开头：

# 正确
ggplot(data, aes(x, y)) +
  geom_point()

# 错误（+ 不能在行首）
ggplot(data, aes(x, y))
  + geom_point()

23.4 进阶图表类型

23.4.1 小提琴图与箱线图组合

小提琴图（Violin Plot）是箱线图的”升级版”，它结合了箱线图和密度图的特点，能够同时展示数据的分布形态和统计特征。对于生态学数据而言，小提琴图特别适合展示物种丰富度、功能多样性等指标在不同处理或环境梯度间的分布差异。

Tip小提琴图参数说明
  • trim = TRUE：将小提琴尾部裁剪到数据范围内；trim = FALSE 保留完整尾部
  • 小提琴的宽度表示该值附近的观测密度，宽度越宽表示数据越集中
  • 叠加箱线图时用 width 控制宽度
  • 若只想显示均值点，可添加 stat_summary(fun = "mean", geom = "point", color = "red")

23.4.2 分组小提琴图

23.4.3 山脊图（ggridges）

山脊图（Ridgeline Plot）是一种展示多组数据分布变化的可视化方式，特别适合展示时间序列数据的分布变化（如季节动态）或环境梯度上的物种分布变化。ggridges 包提供了便捷的山脊图绘制功能。

Tipggridges 进阶用法
  • rel_min_height：控制尾部裁剪，0.01 表示裁剪掉密度低于最大值 1% 的尾部
  • 排序：山脊图默认按 y 轴顺序排列，可通过 factor(season, levels = c("春","夏","秋","冬")) 控制顺序
  • 分面：facet_wrap(~land_use) 可进一步按土地利用类型分面
  • 叠加点：geom_jitter() 可添加原始数据点

23.4.5 等高线图（geom_contour）

等高线图用于展示三维空间数据（两个连续变量 + 一个响应变量）的分布模式。在生态学中，等高线图常用于展示物种丰富度、生物量或土壤性质在空间或环境梯度上的分布。

Note等高线图类型对比
函数 用途 适用场景
geom_contour() 线状等高线 等值线分析
geom_contour_filled() 填充等高线 空间分布热力图
geom_density_2d() 二维密度等高线 点密度分布

当没有现成的 z 值时，可用 stat_density_2d() 从 x-y 坐标自动计算密度。

23.4.7 网络图（ggraph）

23.5 配色方案详解

23.5.1 ColorBrewer 色板

ColorBrewer 是专门为地图和分级统计图设计的配色方案，提供三大类配色：

  • ** Sequential（顺序型）**：从浅到深的单一色相，适合展示从低到高的连续变量
  • ** Diverging（分类型/离异型）**：两端为不同色相，中间为中性色，适合展示相对于参考值的高低
  • ** Qualitative（离散型）**：多种色相，适合展示分类变量
TipColorBrewer 常用色板

离散型（Qualitative）： - "Set1" / "Set2" / "Set3"：适合分类数据 - "Dark2"：暗色调，适合有背景的图 - "Paired"：成对色板，适合有内在顺序的分类

顺序型（Sequential）： - "Blues" / "Greens" / "Oranges" / "Reds"：单一色相渐变 - "viridis"：色盲友好首选

在 ggplot2 中使用：scale_fill_brewer(palette = "Set2") 或 scale_color_brewer(palette = "Set2")

23.5.2 分面展示不同配色方案

23.5.3 viridis 变体对比

viridis 色板是一套对色盲友好、在灰度打印时仍可区分的配色方案，包含 5 种变体：

变体 色相 特点 推荐场景
"A"（magma） 白→红→紫→黑 亮度递增 夜间模式背景
"B"（inferno） 黄→橙→红→紫→黑 亮度递增 高对比度
"C"（plasma） 白→黄→橙→粉→紫 无绿色 绿-红色盲友好
"D"（viridis） 黄→绿→蓝→紫 默认均衡 通用推荐
"E"（cividis） 深蓝→青→黄 完全色盲友好 无障碍设计
Tipviridis 连续 vs 离散
  • 连续变量：scale_color_viridis_c(option = "D") / scale_fill_viridis_c()
  • 离散变量（≤ 8 类）：scale_color_viridis_d(option = "D") / scale_fill_viridis_d()
  • 离散变量（> 8 类）：viridis 不推荐超过 8 个类别，可考虑 ColorBrewer 的离散色板

23.5.4 色盲友好设计原则

  1. 避免红-绿对比（最常见的色盲类型）
  2. 避免单独的色相区分，添加亮度/纹理辅助
  3. 在论文提交前，用色盲模拟器（如 Coblis、Color Oracle）检查
  4. viridis 和 cividis 专为色盲友好设计，可优先选用

23.6 交互式可视化（plotly）

plotly 是 R 语言中功能强大的交互式可视化库，支持直接转换 ggplot2 图表为交互式版本。

23.6.1 ggplotly 转换

将 ggplot2 图表直接转换为 plotly 交互式图表，只需用 ggplotly() 包装即可：

Tipggplotly 交互功能

ggplotly() 是 plotly 包提供的强大函数，可以将静态的 ggplot2 图表一键转换为交互式图表，无需重写代码。这对于探索性数据分析（EDA）和数据展示非常有用，尤其是在需要向非技术人员展示复杂数据时。

核心交互功能：

  1. 悬停提示（Hover Tooltip）：鼠标移到数据点上自动显示原始数据值，可通过 tooltip 参数自定义显示内容（如 tooltip = c("x", "y", "color")）
  2. 缩放（Zoom）：可框选放大特定区域，双击恢复原始视图，适合查看密集数据点的细节
  3. 平移（Pan）：拖动图表移动查看范围，配合缩放功能可以灵活探索数据
  4. 图例交互：点击图例项可隐藏/显示对应的数据系列，双击可单独显示某一系列
  5. 导出：可导出为 PNG/SVG/PDF 格式，方便分享和报告使用

生态学应用场景：

  • 物种分布探索：在散点图中悬停查看每个样点的物种名称、坐标和环境因子
  • 时间序列分析：缩放查看特定时间段的变化趋势，识别异常值
  • 多变量关系：通过图例交互对比不同处理组或生境类型的差异
  • 数据质量检查：快速定位异常数据点并查看其详细信息

使用技巧：

# 自定义悬停信息
p <- ggplot(data, aes(x = var1, y = var2, text = paste("样点:", site_id))) +
  geom_point()
ggplotly(p, tooltip = "text")  # 只显示自定义的 text 内容

# 禁用某些交互功能
ggplotly(p) %>%
  layout(dragmode = FALSE)  # 禁用拖动

与静态图表的对比：

特性 ggplot2（静态） ggplotly（交互）
数据探索 需要多次调整代码重新绘图 直接在图表上交互探索
细节查看 需要放大坐标轴范围 框选缩放即可
数据标注 需要手动添加 geom_text() 悬停自动显示
文件大小 小（PNG/PDF） 大（HTML，包含 JavaScript）
适用场景 论文发表、打印报告 在线展示、数据探索

注意事项：

  • ggplotly() 不支持所有 ggplot2 的几何对象（如 geom_violin() 可能显示异常）
  • 转换后的图表是 HTML widget，需要在支持 JavaScript 的环境中查看（如浏览器、RStudio Viewer）
  • 某些复杂的 ggplot2 主题设置可能在转换后丢失，需要手动调整

23.6.2 性能提示

交互式图表虽然功能强大，但在处理大型数据集时可能遇到性能瓶颈。理解性能限制和优化策略，可以避免浏览器卡顿或渲染失败。

性能瓶颈的根源：

  1. 数据量过大：ggplotly() 将每个数据点转换为 SVG 元素，超过 10,000 个点时浏览器渲染压力大
  2. JavaScript 开销：交互功能依赖 JavaScript 库（Plotly.js），初始化和事件监听消耗资源
  3. HTML 文件体积：交互式图表嵌入 HTML 时，数据以 JSON 格式存储，大数据集导致文件体积膨胀（可达数十 MB）

性能优化策略：

策略 适用场景 实现方法 效果
数据采样 探索性分析，不需要全部数据点 dplyr::sample_n(data, 5000) 显著提升速度
数据聚合 时间序列或空间数据 按时间段或网格聚合 减少数据点数量
使用 plot_ly() 需要精细控制交互 直接用 plot_ly() 而非 ggplotly() 避免转换开销
WebGL 渲染 超大数据集（> 100,000 点） toWebGL() 或 plot_ly(..., type = "scattergl") GPU 加速渲染
静态图表 最终报告或论文 保持 ggplot2 静态图 无性能问题

实战示例——处理大型数据集：

# 场景：10 万条土壤采样数据，需要交互式探索

# 方法 1：随机采样（适合探索性分析）
large_data <- read_csv("soil_samples_100k.csv")
sampled_data <- large_data |> sample_n(5000)
p <- ggplot(sampled_data, aes(x = ph, y = organic_carbon)) + geom_point()
ggplotly(p)

# 方法 2：空间聚合（适合空间数据）
aggregated_data <- large_data |>
  mutate(
    lon_bin = round(longitude, 1),  # 聚合到 0.1° 网格
    lat_bin = round(latitude, 1)
  ) |>
  group_by(lon_bin, lat_bin) |>
  summarise(mean_ph = mean(ph), n = n(), .groups = "drop")

plot_ly(aggregated_data, x = ~lon_bin, y = ~lat_bin, z = ~mean_ph,
        type = "heatmap")

# 方法 3：WebGL 加速（适合必须显示全部数据的场景）
plot_ly(large_data, x = ~ph, y = ~organic_carbon,
        type = "scattergl",  # 使用 WebGL 渲染
        mode = "markers")

性能基准测试：

数据点数 ggplotly() 渲染时间 plot_ly() 渲染时间 scattergl 渲染时间
1,000 < 1 秒 < 1 秒 < 1 秒
10,000 3-5 秒 2-3 秒 < 1 秒
50,000 15-30 秒 10-15 秒 2-3 秒
100,000 > 60 秒（可能失败） > 30 秒 5-8 秒

生态学案例：某研究团队分析全球森林样地数据（50 万条记录），最初用 ggplotly() 转换散点图时浏览器崩溃。改用以下策略后成功： 1. 按 1° × 1° 网格聚合数据，将 50 万点减少到 5000 个网格 2. 用 plot_ly() 绘制热力图展示空间分布 3. 对于需要查看原始数据点的区域，提供区域筛选功能后再用 scattergl 显示

何时不应使用交互式图表：

  • 论文发表（期刊通常只接受静态图）
  • 打印报告（交互功能无法体现）
  • 数据保密（HTML 文件包含原始数据，易泄露）
  • 网络受限环境（需要加载外部 JavaScript 库）

最佳实践：

  1. 探索阶段：用交互式图表快速发现模式
  2. 分析阶段：根据数据量选择合适的渲染方式
  3. 展示阶段：根据受众和媒介选择静态或交互式图表

23.7.3 3D 图表的问题

3D 图表看似酷炫，但在科研论文中存在严重问题：

  1. 透视失真：靠近观察者的对象显得更大
  2. 遮挡问题：前面的数据遮挡后面的数据
  3. 角度依赖：不同观看角度给出不同印象
  4. 难以精确读数：无法准确读取数值

替代方案：使用分面、颜色编码或等高线图替代 3D 散点图

Warning3D 图表替代方案
3D 问题 替代方案
三维散点图 二维散点图 + 颜色/大小编码第三变量
3D 曲面图 等高线填充图
3D 柱状图 分组柱状图或热力图
3D 折线图 多面板折线图

对于必须使用 3D 的场景（如蛋白质结构），建议使用专业的可视化软件（如 PyMOL、Chimera），而非 ggplot2/plotly。

23.7.4 饼图的替代方案

饼图在科研论文中常常不是最佳选择，原因如下：

  1. 人眼对角度的感知不如长度精确
  2. 当类别接近时难以区分大小
  3. 不适合展示负值或零值
  4. 多类别时难以阅读

替代方案：水平柱状图更适合展示分类数据的比较

Tip饼图替代选择

如果必须使用饼图，建议： 1. 类别不超过 5 个 2. 使用水平柱状图替代 3. 添加数据标签 4. 避免 3D 饼图（扭曲角度）

在生态学论文中，饼图适合展示各类群相对多度等简单比例数据。

23.8 科研论文级图表

23.8.1 自定义主题

ggplot2 提供了多个内置主题（theme_gray()、theme_bw()、theme_minimal() 等），但科研论文通常需要更精细的控制。自定义主题可以确保所有图表风格统一，符合期刊要求。

为什么需要自定义主题？

  1. 期刊要求：不同期刊对图表格式有不同要求（字体大小、网格线、背景色等）
  2. 风格统一：论文中所有图表应保持一致的视觉风格
  3. 可复用性：定义一次主题，应用到所有图表，避免重复代码
  4. 品牌识别：实验室或课题组可以建立统一的视觉识别系统

ggplot2 主题系统的层级结构

ggplot2 的主题系统采用继承机制，可以基于现有主题进行修改：

theme_gray()（默认）
  ├─ theme_bw()（黑白主题）
  ├─ theme_minimal()（极简主题）
  ├─ theme_classic()（经典主题，无网格线）
  └─ theme_void()（空白主题）

Tip主题设计最佳实践
  1. 字体大小：基础字号 10-12pt，标题比正文大 2pt

23.8.2 常见问题

Q: 如何修改单个图表的主题而不影响全局？

A: 在 ggplot() 后添加 + theme_xxx()，会覆盖全局设置。

Q: 如何在自定义主题中使用中文字体？

A: 设置 base_family 参数，如 theme_paper(base_family = "SimSun")（需要系统安装该字体）。

Q: 主题设置不生效怎么办？

A: 确保 theme() 在 theme_xxx() 之后调用，后者会覆盖前者的设置。

23.8.3 多面板组合图

科研论文中常需要将多个相关图表组合成一个复合图（multi-panel figure），以节省版面并便于对比。patchwork 包提供了简洁优雅的图表组合语法，是 ggplot2 生态系统中最强大的组合工具。

为什么需要多面板组合图？

  1. 节省版面：期刊通常限制图表数量，组合图可以在一个图号下展示多个子图
  2. 便于对比：相关图表放在一起，读者可以直接对比
  3. 逻辑清晰：按逻辑顺序排列子图，讲述完整的数据故事
  4. 符合规范：多数期刊要求复合图使用 (a)、(b)、(c) 等标签

图例处理： 1. 统一图例：相同分组变量的子图共用一个图例 2. 位置合理：图例放在不遮挡数据的位置（通常右侧或底部） 3. 简洁明了：图例标签简短清晰

23.8.4 常见问题

Q: 如何调整子图之间的间距？

A: 使用 plot_layout(widths = ..., heights = ...) 或在 theme() 中设置 plot.margin。

Q: 子图的图例大小不一致怎么办？

A: 使用 plot_layout(guides = "collect") 统一图例，或在每个子图中用 theme(legend.key.size = ...) 统一大小。

Q: 如何在组合图中插入表格？

A: 使用 gridExtra::tableGrob() 将表格转换为 grob 对象，然后用 wrap_elements() 包装后组合。

Q: 组合图导出后标签位置不对？

A: 检查 plot.tag.position 设置，确保使用相对坐标（0-1 之间）而非绝对坐标。

23.10 课后练习

  1. 小提琴图练习：绘制不同土地利用类型的物种丰富度小提琴图，叠加箱线图，并添加均值点

  2. 山脊图练习：使用 ggridges 包绘制不同季节生物量的山脊图，比较分布变化

  3. 等高线图练习：基于 ecology 数据集，绘制海拔与 pH 交互作用下有机碳的等高线填充图

  4. 网络图练习：使用 ggraph 包创建一个简单的物种互作网络，调整布局和节点颜色

  5. 配色方案练习：

    • 比较 scale_fill_brewer(palette = "Set2") 和 scale_fill_viridis_d() 的效果
    • 创建一个自定义配色方案应用于箱线图

  6. 交互式可视化练习：将一个 ggplot2 散点图转换为 plotly 交互式图表，添加自定义悬停提示

  7. 过度绘图修复练习：

    • 生成 2000 个随机数据点
    • 分别使用透明度、2D 密度图、分箱统计三种方法修复过度绘图问题

  8. 综合练习：创建一个 2×3 的组合图，包含：散点图、箱线图、小提琴图、山脊图、热力图、柱状图
