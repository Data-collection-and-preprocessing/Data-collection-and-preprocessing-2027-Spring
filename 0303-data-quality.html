### 数据质量控制

数据质量控制（Quality Control, QC）是确保数据可靠性和可重现性的核心环节。在生态学研究中，野外采样和实验室分析过程中不可避免地会产生异常值、缺失值和录入错误，系统化的质量控制流程能够最大限度地保证数据的科学性。

#### 质量控制的三个层次

**1. 采集阶段控制**：在数据产生源头进行质量把关。包括仪器校准（如 pH 计每日标定）、重复测量（关键样本设置技术重复）、现场记录核查（双人复核采样编号）。某森林生态站在土壤采样时，要求每个样地至少 3 个重复点，并在现场拍照记录采样位置，有效降低了后期数据溯源难度。

**2. 录入阶段控制**：防止人为录入错误。推荐使用数据验证规则（如 Excel 的数据有效性功能限制 pH 范围为 0-14）、双人录入比对、自动化数据导入（避免手工抄写）。研究表明，双人独立录入后比对可将错误率从 5% 降至 0.5% 以下。

**3. 分析阶段控制**：通过统计方法识别异常。包括范围检查（超出生物学合理范围）、逻辑检查（如土壤含水量 > 100%）、离群值检测（箱线图法、Z-score 法）、一致性检查（同一样地的重复测量变异系数是否过大）。

#### R 中的质量控制实现

```r
library(dplyr)
library(ggplot2)

# 示例：土壤理化性质数据质量控制
soil_data <- data.frame(
  sample_id = 1:100,
  pH = c(rnorm(95, 6.5, 0.8), -1, 15, 6.2, 6.8, 7.1),  # 包含异常值
  organic_carbon = c(rnorm(95, 25, 8), -5, 150, 28, 22, 30),
  moisture = c(rnorm(95, 35, 10), 120, 32, 38, 28, 35)
)

# 质量控制流程
soil_qc <- soil_data %>%
  mutate(
    # 1. 范围检查：标记超出合理范围的值
    pH_flag = case_when(
      pH < 3 | pH > 10 ~ "out_of_range",
      is.na(pH) ~ "missing",
      TRUE ~ "valid"
    ),
    organic_carbon_flag = case_when(
      organic_carbon < 0 | organic_carbon > 100 ~ "out_of_range",
      is.na(organic_carbon) ~ "missing",
      TRUE ~ "valid"
    ),
    moisture_flag = case_when(
      moisture < 0 | moisture > 100 ~ "out_of_range",
      is.na(moisture) ~ "missing",
      TRUE ~ "valid"
    )
  ) %>%
  # 2. 将异常值替换为 NA
  mutate(
    pH = if_else(pH_flag == "out_of_range", NA_real_, pH),
    organic_carbon = if_else(organic_carbon_flag == "out_of_range", NA_real_, organic_carbon),
    moisture = if_else(moisture_flag == "out_of_range", NA_real_, moisture)
  )

# 3. 生成质量控制报告
qc_summary <- soil_qc %>%
  summarise(
    total_records = n(),
    pH_valid = sum(pH_flag == "valid"),
    pH_missing_rate = mean(pH_flag != "valid") * 100,
    oc_valid = sum(organic_carbon_flag == "valid"),
    oc_missing_rate = mean(organic_carbon_flag != "valid") * 100
  )

print(qc_summary)

离群值检测方法

除了范围检查，还可使用统计方法识别潜在异常值：

# 箱线图法检测离群值（IQR 方法）
detect_outliers <- function(x) {
  Q1 <- quantile(x, 0.25, na.rm = TRUE)
  Q3 <- quantile(x, 0.75, na.rm = TRUE)
  IQR <- Q3 - Q1
  lower_bound <- Q1 - 1.5 * IQR
  upper_bound <- Q3 + 1.5 * IQR

  x < lower_bound | x > upper_bound
}

soil_qc <- soil_qc %>%
  mutate(
    pH_outlier = detect_outliers(pH),
    oc_outlier = detect_outliers(organic_carbon)
  )

# 可视化离群值
ggplot(soil_qc, aes(x = "pH", y = pH)) +
  geom_boxplot() +
  geom_jitter(aes(color = pH_outlier), width = 0.1) +
  scale_color_manual(values = c("FALSE" = "gray", "TRUE" = "red")) +
  labs(title = "土壤 pH 离群值检测", color = "是否离群值")

## 生态学数据质量控制案例：马尾松林调查

下面通过一个完整的案例，演示马尾松林土壤调查数据质量控制的全流程。

### 案例背景

某研究团队在广西南宁某马尾松林开展土壤调查，于2027年3月分10个样地、3个深度采集了100个土壤样品。原始数据存在多种质量问题，需要进行系统性的数据清洗和质量控制。

**研究地点与采样设计**：研究区位于广西南宁市武鸣区伊岭岩风景区（23°12'N, 108°23'E），海拔250-350m，属亚热带季风气候。样地选择在马尾松（*Pinus massoniana*）纯林和马尾松-阔叶树混交林中，林龄20-35年。采用系统布点法，在研究区内设置10个20m×20m样地，样地间距不少于100m以保证空间独立性。

**采样方法与实验室分析**：在每个样地中心点，使用土钻采集0-10cm、10-20cm、20-40cm三个深度的土壤样品，每个深度3个技术重复后混合。土壤样品带回实验室后，去除根系和石块，过2mm筛，分为两份：一份风干后用于pH和粒径分析，一份鲜样4°C保存用于微生物分析。土壤pH采用水土比2.5:1的电位法测定，有机碳采用重铬酸钾氧化-外加热法测定，全氮采用凯氏定氮法测定。

**数据录入与初步检查**：实验室分析完成后，由两名研究助理分别独立录入数据到Excel表格中。初步检查发现，两人录入的数据存在约5%的不一致，主要集中在小数点位置和数值抄写错误。此外，部分样品因仪器故障（pH计电极老化、消解管破裂）导致测定失败，产生了缺失值。还有个别样品的测定结果明显超出合理范围（如pH为-0.5或15.2），怀疑是仪器故障或记录笔误。

**质量控制的必要性**：这些数据质量问题如果不加以识别和处理，将直接影响后续的统计分析和科学结论。例如，异常的pH值（-0.5）会严重偏离真实值，导致均值和标准差的估计偏差；缺失值如果处理不当（如简单删除含缺失值的样地），会降低统计检验的效力；录入错误（如有机碳999 g/kg）如果未被发现，会被误认为是真实的极端值。因此，系统化的数据质量控制流程是确保研究可靠性的关键步骤。

## 数据采样与代表性评估

在生态学研究中，我们很少能够测量整个种群或生态系统的所有个体。因此，**采样**（sampling）成为获取数据的主要方式。采样的核心问题是：如何用有限的样本代表整体？一个设计不当的采样方案可能导致**采样偏差**（sampling bias），使得样本无法真实反映总体特征，进而导致错误的科学结论。

### 采样偏差与代表性

**采样偏差**是指样本的统计特征系统性地偏离总体特征。常见的偏差来源包括：

1. **空间偏差**：只在容易到达的地点采样（如道路附近），忽略了难以到达但可能具有不同特征的区域
2. **时间偏差**：只在特定时间段采样，忽略了季节或昼夜变化
3. **选择偏差**：有意或无意地选择了特定类型的样本（如只选择健康的植株）

**代表性**（representativeness）是指样本在多大程度上能够反映总体的特征。一个具有代表性的样本应该：

- 覆盖总体的主要变异范围
- 各类型样本的比例与总体一致
- 样本间相互独立，避免空间或时间上的自相关

### 常见采样方法

**1. 简单随机采样（Simple Random Sampling）**

每个个体被抽中的概率相等。适用于总体较为均质的情况。

**优点**：无偏估计，统计推断简单
**缺点**：当总体异质性高时，可能遗漏重要的亚群体

**2. 分层采样（Stratified Sampling）**

将总体划分为若干层（strata），在每层内进行随机采样。适用于总体存在明显分组的情况。

**优点**：确保各亚群体都被代表，提高估计精度
**缺点**：需要事先了解总体的分层结构

**3. 系统采样（Systematic Sampling）**

按固定间隔抽取样本（如每隔10米设置一个样点）。

**优点**：操作简便，空间覆盖均匀
**缺点**：如果总体存在周期性模式，可能产生偏差

- **系统采样**：空间覆盖均匀，但如果总体按区域排列，可能恰好每层抽到相同数量

### 样本量计算

样本量的确定需要平衡统计精度和调查成本。样本量过小会导致估计不精确，过大则浪费资源。

**样本量计算公式**（估计总体均值）：

$$
n = \frac{Z^2 \sigma^2}{E^2}
$$

其中：
- $n$：所需样本量
- $Z$：置信水平对应的Z值（95%置信水平时Z=1.96）
- $\sigma$：总体标准差（通常用预调查估计）
- $E$：允许误差（期望的估计精度）

### 生态学应用案例：马尾松林样地采样设计

**研究背景**：某团队计划在广西木论国家级自然保护区开展马尾松林生物量调查，研究区面积约500公顷，地形复杂，包含山脊、山坡和山谷三种地形。

**采样设计挑战**：

1. **地形异质性高**：不同地形的马尾松生长差异显著
2. **调查成本有限**：只能设置30个样地
3. **代表性要求**：需要准确估计整个研究区的平均生物量

**解决方案**：采用分层采样

1. **分层依据**：根据地形将研究区划分为三层
   - 山脊（占比30%）：设置9个样地
   - 山坡（占比50%）：设置15个样地
   - 山谷（占比20%）：设置6个样地

2. **样地布设**：在每层内采用随机布点，确保样地间距≥100米

3. **验证代表性**：
   - 样地海拔范围（450-850m）覆盖了研究区海拔范围（400-900m）
   - 样地坡度范围（5-35°）覆盖了研究区坡度范围（0-40°）
   - 样地林龄范围（15-40年）覆盖了研究区林龄范围（10-45年）

**结果**：该采样方案使得生物量估计的标准误降低了35%（相比简单随机采样），同时确保了各地形类型都有足够的样本量进行统计分析。

**关键启示**：

- 当研究区存在明显的环境梯度或分组时，分层采样优于简单随机采样
- 样本量分配应与各层的面积比例一致（比例分配）
- 采样前的预调查（pilot study）有助于确定合理的分层方案和样本量

## 数据版本管理与可追溯性

在科研项目中，数据会经历多次修改：清洗异常值、添加新变量、修正错误记录......如果没有版本管理，你可能会遇到这些问题：

- "这个结果是用哪个版本的数据算的？"
- "上周的分析结果为什么今天重现不了？"
- "谁改了这个数据？为什么改？"

**数据版本管理**（data version control）和**可追溯性**（traceability）是确保研究可重现性的关键。

### 为什么需要数据版本管理？

**1. 可重现性**：科学研究的基石是结果可重现。如果数据版本混乱，即使代码完全相同，也可能得到不同的结果。

**2. 协作需求**：多人协作时，需要知道谁在什么时候修改了数据，避免相互覆盖。

**3. 错误回溯**：发现数据错误时，能够追溯到错误引入的时间点，快速定位问题。

**4. 审稿要求**：越来越多的期刊要求提供数据处理的完整记录，版本管理是满足这一要求的基础。

### 数据版本管理的三个层次

| 层次 | 方法 | 适用场景 |
|:---|:---|:---|
| 基础 | 文件命名 + 日期 | 个人项目，数据变动少 |
| 中级 | Git + 数据快照 | 团队协作，数据变动频繁 |
| 高级 | DVC / LakeFS | 大数据集，需要专业数据管理 |

### 生态学应用案例：长期生态监测数据管理

**研究背景**：广西木论国家级自然保护区自2010年起开展长期森林动态监测，每5年进行一次全面复查。监测内容包括树木生长、死亡、新增个体等。

**数据管理挑战**：

1. **时间跨度长**：项目持续15年以上，数据累积量大
2. **人员更替**：不同批次的研究生参与数据采集和整理
3. **数据修正**：发现历史数据错误时需要追溯和修正
4. **可重现性**：需要确保历史分析结果可以重现

**版本管理方案**：

项目目录结构： mulun-forest-dynamics/ ├── data/ │ ├── raw/ # 原始数据（只读） │ │ ├── census_2010.csv │ │ ├── census_2015.csv │ │ └── census_2020.csv │ ├── processed/ # 处理后数据 │ │ ├── trees_cleaned_2010.csv │ │ ├── trees_cleaned_2015.csv │ │ └── trees_cleaned_2020.csv │ ├── snapshots/ # 数据快照 │ └── CHANGELOG.md # 变更日志 ├── scripts/ │ ├── 01-data-cleaning-2010.R │ ├── 02-data-cleaning-2015.R │ └── 03-data-cleaning-2020.R └── .git/ # Git版本控制 ```

关键实践：

  1. 原始数据不可变：data/raw/ 目录下的文件从不修改，所有处理都在脚本中完成
  2. 脚本化处理：每次普查的数据清洗都有独立的R脚本，确保可重现
  3. Git提交规范：每次数据更新都提交到Git，提交信息包含日期和修改内容
  4. 定期快照：每次发表论文前创建数据快照，确保论文结果可追溯

成果：

  • 2023年审稿人要求重现2015年论文的分析结果，团队通过Git回退到2015年的数据版本，成功重现了所有图表
  • 2024年发现2010年数据中有3棵树的坐标记录错误，通过变更日志追溯到录入时间，联系当时的研究生确认并修正
  • 项目数据已被其他5个研究团队引用，数据管理的规范性获得高度评价

关键启示：

  • 长期项目必须建立系统化的数据版本管理制度
  • 原始数据不可变原则是数据管理的基石
  • 详细的变更日志是团队协作和数据追溯的关键
  • 定期备份和快照是数据安全的保障

19.2 数据质量综合案例：森林生态系统调查

本案例展示一个完整的森林生态系统调查数据质量控制流程，从数据采集到最终报告生成。

19.2.1 案例背景

研究项目：广西大明山国家级自然保护区森林碳储量调查

调查时间：2027年3月-5月

调查范围： - 海拔范围：800-1200米 - 植被类型：常绿阔叶林、针阔混交林、马尾松林 - 样地数量：50个20m×20m样地 - 调查指标：树种、胸径、树高、冠幅、健康状况

数据采集过程： - 野外调查：3个调查小组，每组2人 - 数据录入：2名研究助理独立录入后比对 - 实验室分析：土壤样品的理化性质测定

已知问题： 1. 部分样地因天气原因未能完成调查（缺失值） 2. 不同调查员的测量标准存在差异（系统误差） 3. 数据录入时存在笔误（如胸径120cm误录为1200cm） 4. 部分树种鉴定存在疑问（分类不一致） 5. 土壤样品因仪器故障导致部分指标缺失

19.2.2 完整清洗流程

19.2.3 关键发现与建议

数据质量总结：

  1. 完整性：清洗后数据的综合完整率达到93.5%，满足分析要求
  2. 准确性：识别并修正了43个异常值，其中18个为明显的录入错误
  3. 一致性：统一了树种和健康状况的分类标准
  4. 可追溯性：所有清洗步骤均有详细记录，可完全重现

主要问题：

  1. 录入错误：发现18个明显的录入错误（如胸径1200cm），占比1.8%
  2. 缺失数据：约6.5%的记录存在缺失值，主要集中在3个样地（天气原因）
  3. 分类不一致：10%的树木未能鉴定到种，标记为”未鉴定”

改进建议：

  1. 数据采集阶段：
    • 使用电子数据采集器（如平板电脑），减少手工录入
    • 设置实时数据验证规则，现场发现异常值
    • 加强调查员培训，统一测量标准
  2. 数据录入阶段：
    • 实施双人独立录入后比对制度
    • 使用数据验证规则（如Excel数据有效性）
    • 定期进行数据质量检查
  3. 数据管理阶段：
    • 建立标准化的数据清洗流程
    • 使用Git进行版本控制
    • 定期备份原始数据和处理后数据
  4. 后续工作：
    • 对”未鉴定”树种进行补充鉴定（咨询植物分类专家）
    • 对缺失数据较多的3个样地进行补充调查
    • 建立长期监测数据库，为后续复查做准备

数据可用性声明：

经过系统化的质量控制，本数据集可用于： - 森林碳储量估算 - 树种组成分析 - 径级结构分析 - 健康状况评估

不建议用于： - 精确的单木生长模型（部分树木缺少历史数据） - 稀有树种分析（未鉴定树种比例较高）

19.3 课后练习

  1. 元数据编写：为你的课程项目数据编写一份完整的元数据文件（README.md），包含数据集名称、采集时间、变量说明、数据处理记录等内容。

  2. 数据验证函数：编写一个数据验证函数，检查你的数据是否满足以下条件：

    • 数值变量在合理范围内（如 pH 0-14，有机碳 0-100 g/kg）
    • 分类变量水平一致（无拼写错误、大小写不一致）
    • 日期格式统一
    • 粒径百分比之和不超过100%

  3. 异常值检测：使用 Z 分数法和 IQR 法分别检测你数据中的异常值，比较两种方法的结果差异，并结合领域知识判断异常值的合理性。

  4. 缺失值处理比较：对你的数据分别使用列表删除法、均值填充法和 KNN 填充法，比较处理后的数据分布差异，评估不同方法对你分析结果的影响。

  5. 完整清洗流程：选择一个包含多种数据质量问题的数据集，完整执行检测→诊断→修正→验证的清洗流程，并编写数据质量报告。
