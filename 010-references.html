27.1.1 数据采集方法

生态学研究的数据采集方法多种多样，根据研究对象和研究目的的不同，需要选择合适的采集方法。常见的数据采集方法包括样方法、样线法、样带法、访问调查法、传感器监测法和遥感监测法等。

样方法是生态学中最经典的数据采集方法，适用于植物群落调查、土壤采样等工作。在喀斯特生态系统中，由于地形复杂、地表破碎，样方法需要结合地形特点进行灵活布设。通常选取典型地段设置样方，样方大小根据研究对象确定——草本群落常用1平方米，灌木群落常用10平方米，森林群落常用100平方米或更大。调查时记录物种名称、多度、盖度、频度、株高等指标，同时采集土壤样品装入密封袋，记录采样深度和样品编号。

样线法适合研究植物群落的水平结构变化。在喀斯特地区，可沿不同海拔或土地利用类型设置样线，记录物种组成随环境梯度的变化。样线长度通常为100米至1000米，每隔一定距离设置观测点，记录物种分布情况。

传感器自动采集是现代生态学研究的重要手段。自动气象站可以连续记录温度、湿度、降水等气象数据；土壤水分传感器能够定点监测土壤含水量动态变化；根系生长监测系统可以跟踪植物根系的发育过程。这些自动化设备大大提高了数据采集的时空分辨率。

遥感监测则能获取大尺度的植被覆盖和土地利用信息。通过卫星影像可以分析喀斯特地区的植被覆盖度变化、石漠化发展趋势等宏观生态问题。

无论采用何种方法，数据采集前都应制定详细的采样方案，明确采样目的、对象、数量、时间、地点等要素。采样过程中要做好现场记录，确保数据的可追溯性。采集完成后，需及时整理数据，建立规范的数据库结构，便于后续的分析处理。

数据预处理与清洗是数据分析工作中最耗时但也最重要的环节。在生态学研究中，我们从野外采集的原始数据往往存在各种问题：缺失值、异常值、格式不统一、单位混乱等。这些数据质量问题如果不及时处理，会严重影响后续统计分析的准确性，甚至导致错误的结论。数据清洗的目的就是识别并纠正这些问题，使数据达到可用于分析的标准。

生态学野外数据中常见的质量问题包括：监测样点数据缺失（如某次调查中漏测了某个样方的物种多度）、仪器测量异常值（如溶氧仪读数超出合理范围）、录入时的人为错误（如将物种名拼写错误或小数点位置错误）、数据格式不一致（如日期格式混用”2023-01-15”和”2023/1/15”）、单位不统一（如将厘米和米混用）等。

数据清洗的一般流程是：先对数据进行探索性检查，了解数据的基本结构和分布特征；然后识别问题数据，判断是缺失值、异常值还是错误值；接着针对不同问题采取相应的处理方法，常用的处理方式包括删除、替换、插值等；最后验证清洗结果，确保数据质量满足分析需求。

在R语言中，tidyverse生态提供了丰富的数据清洗函数。tidyr包专门用于处理缺失值和变量重塑，dplyr包则擅长进行数据的筛选、排序、分组汇总等操作。这两个包的语法简洁一致，学习成本低，深受生态学研究者喜爱。

下面我们以喀斯特石山区土壤养分调查数据为例，演示如何对原始数据进行系统的清洗处理。

library(tidyverse)

# 创建喀斯特土壤养分调查示例数据（包含常见的脏数据问题）
soil_data <- tibble(
  样地编号 = c("KST01", "KST02", "KST03", "KST04", "KST05", "KST06", "KST07", "KST08"),
  有机质 = c(4.52, NA, 3.81, -0.5, 5.23, NA, 4.18, 3.95),
  全氮 = c(0.23, 0.18, NA, 0.21, 0.25, 0.19, 0.22, NA),
  pH值 = c(5.2, 5.8, "6.2", 999, 5.5, 6.1, 5.9, 5.6),
  采样日期 = c("2023-03-15", "2023/3/16", "2023-03-17", "2023-03-18", 
                "2023-03-19", "2023-03-20", "2023-03-21", "2023-03-22")
)

cat("【原始数据概览】\n")
print(soil_data)
cat("\n【数据类型检查】\n")
str(soil_data)

# 步骤1：统一日期格式并转换为标准格式
soil_clean <- soil_data %>%
  mutate(采样日期 = parse_date(采样日期, format = c("%Y-%m-%d", "%Y/%m/%d")))

# 步骤2：处理pH值中的异常值（999为缺失标记），并转换为数值型
soil_clean <- soil_clean %>%
  mutate(pH值 = ifelse(as.numeric(pH值) > 14 | as.numeric(pH值) < 0, NA, as.numeric(pH值)))

# 步骤3：处理有机质中的异常负值（物理不可能），替换为NA
soil_clean <- soil_clean %>%
  mutate(有机质 = ifelse(有机质 < 0, NA, 有机质))

# 步骤4：识别并检查全氮的异常值（使用四分位距法）
Q1 <- quantile(soil_clean$全氮, 0.25, na.rm = TRUE)
Q3 <- quantile(soil_clean$全氮, 0.75, na.rm = TRUE)
IQR_val <- Q3 - Q1
lower_bound <- Q1 - 1.5 * IQR_val
upper_bound <- Q3 + 1.5 * IQR_val
cat("\n【全氮异常值检测边界】\n")
cat(sprintf("下界: %.3f, 上界: %.3f\n", lower_bound, upper_bound))

# 步骤5：使用该样地内其他样点的中位数填充缺失值
soil_clean <- soil_clean %>%
  mutate(有机质 = coalesce(有机质, median(有机质, na.rm = TRUE)),
          全氮 = coalesce(全氮, median(全氮, na.rm = TRUE)),
          pH值 = coalesce(pH值, median(pH值, na.rm = TRUE)))

cat("\n【清洗后数据】\n")
print(soil_clean)
cat("\n【清洗后缺失值统计】\n")
print(colSums(is.na(soil_clean)))

### R 语言编程

R 语言是生态学数据分析领域最受欢迎的编程语言之一。与传统统计软件相比，R 拥有强大的扩展包生态系统，其中 tidyverse 是数据处理的核心工具包。tidyverse 由 ggplot2、dplyr、tidyr、readr 等多个扩展包组成，采用一致的语法设计理念，让数据清洗和转换变得直观高效。

在生态学研究中，我们经常需要处理从野外采集的多源数据。R 语言的优势在于：代码可重复执行、数据处理流程透明、支持从数据导入到可视化分析的全流程操作。对于土壤生态学研究而言，使用 R 可以高效处理大量的土壤理化性质数据、微生物群落数据以及环境因子数据，实现从"脏数据"到"干净数据集"的快速转换。

使用 R 进行数据预处理的基本思路是：首先读取原始数据，然后通过管道操作符（%>%）将数据依次传递给筛选、变换、汇总等函数，最终输出整理好的数据框。这种"流水线"式的数据处理方式不仅代码简洁，还便于后续检查和修改。掌握 R 语言编程将为生态学研究提供强大的数据处理能力，是现代生态学研究者的必备技能。

  ph = c(5.2, NA, 6.1, 5.8, NA, 5.5, 6.3, 5.9, NA, 6.0,

在生态学研究中，统计分析与建模是将原始数据转化为科学结论的核心环节。当你完成了数据清洗、缺失值处理和变量转换后，接下来的工作就是从数据中提取规律、验证假设，甚至预测未来趋势。统计分析不仅仅是”跑个检验”那么简单，它要求你明确研究问题、选择合适的方法，并正确解读结果。

生态学数据通常具有层次性（如样点嵌套于样地）、非独立性（如时间序列）和非线性关系（如物种-面积曲线）。因此，传统的一元方差分析或简单线性回归往往不够用。你需要掌握更灵活的工具：广义线性模型（GLM）处理非正态数据、混合效应模型处理层次结构、空间自相关分析处理地理数据。

建模的第一步是探索性分析。散点图矩阵、相关系数矩阵、箱线图能帮你快速了解变量间的关系。对于非线性关系，可以考虑对数变换、二次项，或使用样条函数。模型选择时，既要避免欠拟合（模型太简单，解释力不足），也要警惕过拟合（模型太复杂，对新数据预测能力差）。赤池信息准则（AIC）和交叉验证是常用的模型比较工具。

拿到结果后，别忘了检验模型假设。残差的正态性、方差齐性、异常值检测都是必要步骤。如果假设不满足，可以尝试数据变换或改用更稳健的方法。最终，你的分析应该回答最初的科学问题，而不是机械地输出一堆p值。

27.1.2 喀斯特区土壤有机碳储量与环境因子的关系建模

下面我们用西南喀斯特地区一个森林样地的土壤调查数据，演示如何用tidyverse进行探索性分析和多元回归建模。数据包含不同样点的有机碳含量（SOC）、pH值、粘粒含量、含水量和海拔高度。

library(tidyverse)

# 模拟喀斯特森林土壤调查数据
set.seed(2024)
n <- 60
karst_soil <- tibble(
  site_id = paste0("K", sprintf("%03d", 1:n)),
  elevation = round(runif(n, 800, 1600), 1),  # 海拔(m)
  clay_content = round(runif(n, 20, 60), 1),   # 粘粒含量(%)
  soil_moisture = round(runif(n, 25, 55), 1),  # 土壤含水量(%)
  pH = round(runif(n, 4.5, 7.0), 2),           # 土壤pH
  SOC = round(15 + 0.012 * (elevation - 800) + 0.3 * clay_content - 
              2.5 * (pH - 6) + rnorm(n, 0, 3), 2)  # 有机碳(g/kg)
)

# 查看数据概况
cat("=== 喀斯特森林土壤数据概况 ===\n")
summary_stats <- karst_soil %>%
  select(-site_id) %>%
  summarise(across(everything(), list(
    mean = ~mean(., na.rm = TRUE),
    sd = ~sd(., na.rm = TRUE),
    min = ~min(., na.rm = TRUE),
    max = ~max(., na.rm = TRUE)
  ))) %>%
  pivot_longer(everything(), names_to = c("variable", ".value"), 
               names_pattern = "(.+)_(.+)")

print(summary_stats, n = Inf)

# 相关性分析
cat("\n=== 变量间相关系数 ===\n")
cor_matrix <- karst_soil %>%
  select(-site_id) %>%
  cor() %>%
  round(2)
print(cor_matrix)

# 可视化：有机碳与各因子的关系
p1 <- ggplot(karst_soil, aes(x = elevation, y = SOC)) +
  geom_point(alpha = 0.7) +

### 可重复性研究

可重复性研究是现代生态学研究的重要基石。随着数据规模和复杂性的不断增长，如何确保研究成果能够被独立验证和复现，成为了每个研究者必须面对的问题。

**什么是可重复性研究？**简单来说，就是你的数据分析过程能够被他人完整地重建，包括使用相同的数据、相同的代码、相同的环境配置。在土壤生态学和喀斯特生态系统中，由于采样条件的复杂性和数据的时空异质性，可重复性研究显得尤为重要。一份在广西喀斯特地区采集的土壤碳储量数据，如果缺乏完整的处理流程记录，其他研究者可能难以验证你的结论是否可靠。

**为什么可重复性这么重要？**首先，它能提高研究的可信度。其次，它便于团队协作和知识传承。最后，很多期刊现在都要求作者提供可重复的研究代码和数据。想象一下，你的导师或者未来的你自己要重新运行你的分析代码，如果环境配置混乱，那将是一场噩梦。

**如何实现可重复性研究？**最基本的要求是：代码要写清楚、数据要存档、环境要记录。在R语言中，我们可以使用renv包来管理项目依赖，它能记录每个项目使用的R包版本，确保代码在任何环境下都能正常运行。此外，将代码托管到GitHub等版本控制系统上，配合详细的README文档，就能让任何人对你的研究进行独立验证。

下面的代码演示了如何使用renv包管理项目环境，并完成一个标准化的土壤数据处理流程：

```r
# 首先安装并加载renv包（首次使用时需要安装）
if (!require("renv")) install.packages("renv")
library(renv)

# 初始化项目环境
# renv::init()

# 加载所需包
library(tidyverse)

# 创建一个标准化的数据处理函数
process_soil_data <- function(df) {
  df |>
    mutate(total_carbon = carbon_content * sample_depth * bulk_density) |>
    group_by(land_use_type) |>
    summarize(
      mean_carbon = mean(total_carbon, na.rm = TRUE),
      sd_carbon = sd(total_carbon, na.rm = TRUE),
      n = n()
    )
}

# 创建示例数据集（模拟喀斯特地区土壤调查数据）
set.seed(123)
soil_samples <- tibble(
  plot_id = paste0("K", 1:45),
  land_use_type = rep(c("森林", "草地", "农田"), each = 15),
  carbon_content = c(rnorm(15, 3.2, 0.8), rnorm(15, 2.1, 0.6), rnorm(15, 1.3, 0.4)),
  sample_depth = rep(c(0.1, 0.2, 0.3), 15),
  bulk_density = rep(c(1.2, 1.0, 0.9), 15)
)

# 应用数据处理函数
processed_data <- process_soil_data(soil_samples)
print(processed_data)

生态学是研究生物与环境之间相互关系的科学，生态学数据是理解自然生态系统运作规律的基础。在生态学研究中，数据采集与预处理是连接野外观测与科学结论的桥梁。生态学数据具有明显的时空异质性特征——同一物种在不同地点的分布密度可能相差数个数量级，同一地点在不同季节的群落结构也往往差异显著。这种高度变异性要求我们在数据预处理阶段格外谨慎，否则很容易将采样误差或测量偏差误认为真实的生态学格局。

常见的生态学数据类型包括：物种多度数据（计数或估测丰度）、群落调查数据（多物种多样性的综合表征）、环境因子数据（土壤理化性质、气候变量等），以及遥感影像数据（植被指数、土地覆盖等）。不同类型的数据需要不同的预处理策略。例如，物种多度数据常常需要进行对数变换以满足正态性假设；群落数据则可能需要使用Hellinger转化来消除物种缺失值过多带来的问题。在进行多变量分析前，标准化处理几乎是不可或缺的步骤——将不同量纲的变量放在同一尺度上，才能确保分析结果的可靠性。

喀斯特地貌区的土壤生态系统是生态学研究的重要对象。喀斯特地区土层浅薄、岩石裸露率高，土壤养分容易通过地下漏失流失，生态系统极为脆弱。在这类地区开展生态学研究时，数据采集往往面临样方面积受限、土壤样本采集困难等问题。因此，喀斯特土壤生态学研究对数据质量的要求更高，预处理方法的合理性直接影响对生态系统功能恢复效果的评价。

# 生态学数据预处理示例：喀斯特地区土壤养分分析
library(tidyverse)

# 创建模拟的喀斯特土壤养分数据集
set.seed(123)
soil_data <- tibble(
  site_id = paste0("KST-", sprintf("%03d", 1:50)),
  habitat_type = rep(c("原始林", "次生林", "人工林", "撂荒地"), each = 12, length.out = 50),
  soil_depth = rep(c("0-10cm", "10-20cm", "20-40cm"), length.out = 50),
  pH = rnorm(50, mean = 6.5, sd = 0.8),
  organic_matter = rnorm(50, mean = 4.2, sd = 1.5),
  total_N = rnorm(50, mean = 2.1, sd = 0.7),
  available_P = rnorm(50, mean = 8.5, sd = 3.2),
  available_K = rnorm(50, mean = 120, sd = 45)
)

# 数据清洗与标准化
cleaned_data <- soil_data %>%
  mutate(across(where(is.double), ~round(., 2))) %>%
  mutate(
    pH = case_when(pH < 4 ~ NA_real_, pH > 9 ~ NA_real_, TRUE ~ pH),
    organic_matter = ifelse(organic_matter < 0, 0, organic_matter)
  ) %>%
  drop_na()

# Z-score标准化以便比较不同量纲的养分指标
standardized_data <- cleaned_data %>%
  mutate(across(c(organic_matter, total_N, available_P, available_K), 
                ~scale(.)[,1], .names = "{.col}_z"))

# 按生境类型汇总统计
summary_stats <- standardized_data %>%
  group_by(habitat_type) %>%
  summarise(
    样本数 = n(),
    平均有机质 = mean(organic_matter),
    平均全氮 = mean(total_N),
    平均有效磷 = mean(available_P),
    平均有效钾 = mean(available_K),
    .groups = "drop"
  ) %>%
  mutate(across(where(is.double), ~round(., 3)))

print(summary_stats)
